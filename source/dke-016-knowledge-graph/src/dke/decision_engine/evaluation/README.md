# DIE-003 Decision Evaluation Engine

## Purpose

The evaluation package scores alternatives generated by DIE-002 and returns an `EvaluatedDecisionPackage`.
It gives later decision modules a deterministic, auditable ranking of alternatives before final selection.

## Pipeline

`AlternativeDecisionPackage` -> `DecisionEvaluator` -> `ScoringEngine` -> `EvaluationPackageBuilder` -> `EvaluationValidator` -> `EvaluatedDecisionPackage`

## Scoring Criteria

Default criteria weights sum to `1.0`:

- `feasibility`: `0.20`
- `confidence`: `0.15`
- `goal_alignment`: `0.20`
- `constraint_satisfaction`: `0.15`
- `evidence_support`: `0.15`
- `risk_balance`: `0.10`
- `advantage_balance`: `0.05`

All raw scores are normalized between `0.0` and `1.0`; weighted scores are calculated as `score * weight`.

## Recommendation Levels

- `strong`: overall score >= `0.80`
- `moderate`: overall score >= `0.60`
- `weak`: overall score >= `0.40`
- `not_recommended`: overall score < `0.40`

## Example Usage

```python
from decision_engine.core import DIECore
from decision_engine.alternatives import AlternativeGenerator
from decision_engine.evaluation import DecisionEvaluator

decision_package = DIECore().process({
    "query": "Choose a compliant vendor within budget",
    "semantic_results": [{"text": "Vendor A has strong reliability.", "confidence": 0.9}],
})

alternative_package = AlternativeGenerator().generate(decision_package.decision_state)
evaluated_package = DecisionEvaluator().evaluate(alternative_package)
print(evaluated_package.evaluated_alternatives[0].recommendation_level)
```

## Future Expansion

Future modules can add domain-specific criteria, calibrated scoring from historical outcomes, policy-aware penalties,
or explainability adapters while keeping the `EvaluatedDecisionPackage` contract stable.
